──────────────────────────────────────────────────────
 NOTE: WOKWI SIMULATION ISSUE
──────────────────────────────────────────────────────
Wokwi's simulated "Wokwi-GUEST" WiFi network does NOT
support outbound TCP connections on port 1883 to external
MQTT brokers (like broker.hivemq.com). All 3 ESP32 nodes
were compiled and flashed successfully via PlatformIO, but
cannot publish data in the virtual environment.

WORKAROUND: dashboard/backend/node_simulator.py
  - Replaces all 3 Wokwi simulations with Python threads
  - Publishes realistic Bangalore sensor data to HiveMQ
  - Attacker stays NORMAL until dashboard "Trigger" is pressed
  - Run: python node_simulator.py (from dashboard/backend/)

WITH REAL HARDWARE: Just flash the .bin files, update WiFi
credentials in each main.cpp, and the full pipeline works
without any other code changes.

   (repeat for LDR and Attacker nodes)
