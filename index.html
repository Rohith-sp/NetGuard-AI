Problem Statement

IoT devices present an ever-expanding attack surface. Legacy IDS tools lack the granularity to detect protocol-level exploits on resource-constrained embedded networks — and provide zero human-readable explanations.

With billions of IoT endpoints deployed globally, adversaries exploit weak authentication, unencrypted MQTT channels, and predictable traffic patterns. Traditional signature-based IDS miss novel attack variants; anomaly-based systems generate excessive false positives. Critically, no existing lightweight solution combines real-time edge detection with plain-language AI explanations accessible to non-expert operators.

⚡
DoS Flood Attack

Attacker saturates the broker with thousands of MQTT PUBLISH packets per second, exhausting CPU and memory on constrained ESP32 devices and causing legitimate nodes to lose connectivity.

⚠ Existing IDS: Rate-limiting misses short bursts; signature DBs outdated within days
🔁
Replay Attack

Captured legitimate MQTT packets are re-injected to trigger actuators or bypass authentication. High repeat ratio (65%+) with normal individual packet rates makes this invisible to simple threshold monitors.

⚠ Existing IDS: Cannot distinguish replayed vs. authentic identical payloads without temporal analysis
🔍
Slow-Rate Probe / Port Scan

Adversary probes the network with deliberately low-rate packets (45–60/sec) spread across many ports, evading volume-based detection while mapping the entire device topology over minutes.

⚠ Existing IDS: Volume thresholds never triggered; port entropy anomaly undetected without ML
